Founder: Lokesh Role: Data Science & AI Lead Vision: Build an
intelligent, AI-powered Edu-Tech platform that tracks learning,

  -------------------
  1. PROJECT VISION
  -------------------

The goal is to build a scalable Edu-Tech platform that:

-   Manages students, parents, tutors, and admins
-   Tracks sessions and academic activity
-   Evaluates academic and soft skills performance
-   Uses AI for report generation and recommendations
-   Supports analytics, experimentation, and optimization
-   Is scalable, secure, and production-ready

The system must be modular, AI-ready, analytics-friendly, and
extendable.

  -----------------------------------
  5. DATABASE ARCHITECTURE OVERVIEW
  -----------------------------------

The database is divided into logical domains:

A)  Identity & Access
    -   USERS
    -   USER_PROFILES
    -   PARENT_STUDENT_MAP
B)  Academic Structure
    -   GRADES
    -   SUBJECTS
    -   TOPICS
    -   CURRICULUM_MAP
C)  Sessions & Learning
    -   SESSIONS
    -   SESSION_NOTES
    -   ASSIGNMENTS
D)  Performance & Evaluation
    -   ASSESSMENTS
    -   SKILL_METRICS
E)  AI & Reporting
    -   REPORTS
    -   AI_FEEDBACK_LOGS
    -   RECOMMENDATIONS
F)  System & Monitoring
    -   ACTIVITY_LOGS
    -   AUDIT_LOGS

Design Principles: - Clean separation of domains - UUID primary keys -
Indexed foreign keys - JSON for AI outputs - Time-series support for
skill metrics - Normalized structure - Extendable architecture

  ---------------------------
  6. DATA FLOW (HIGH-LEVEL)
  ---------------------------

User registers → Parent linked → Session scheduled → Session completed →
Assessment recorded → Skill metrics evaluated → AI generates report →
Recommendations created → Activity logged

Everything connects logically and sequentially.

  ----------------
  7. AI STRATEGY
  ----------------

OpenAI API will be used for: - Structured report generation -
Personalized recommendations - Feedback summaries - Prompt-based
analytics insights

AI outputs are stored separately in: - REPORTS (JSON format) -
AI_FEEDBACK_LOGS - RECOMMENDATIONS

This ensures auditability and flexibility.

  -----------------------------
  8. SYSTEM DESIGN PRINCIPLES
  -----------------------------

-   Modular architecture
-   Clear separation of concerns
-   Backend handles business logic
-   Frontend handles presentation
-   AI layer isolated from core academic data
-   Logging and auditing for production safety
-   Indexed tables for performance
-   Phased implementation (MVP → Scale)

  9. LONG-TERM SCALABILITY STRATEGY
