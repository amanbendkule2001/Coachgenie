## CORRECTED CODE

# Example 1 - If a student let's downloads a file or multiple file, then the code will somewhat look like this.

When, download button pressed:
  # STEPS
    1. User clicks Download
    2. Frontend sends request to Backend:
        GET /file/123/download

    3. Backend:
        - Validates user token
        - Checks permission
        - Queries DB for file metadata
        - Gets file storage location (S3 URL)

    4. Backend generates signed URL (secure temporary link)

    5. Backend sends signed URL to Frontend

    6. Frontend triggers download directly from storage/CDN

    7. User downloads file

# Example 2 - If a parent asks for a report, then the code will somewhat look like this.

When, get_report or some related action is performed by the User:
  # STEPS

Option A – Simple Report (Quick Response)

1. Parent clicks "Generate Report"
2. Frontend → Backend API
3. Backend validates token
4. Backend fetches required data from DB
5. Backend sends data to AI/Analytics Engine
6. AI processes & returns analysis
7. Backend formats structured report
8. Backend sends JSON to Frontend
9. Frontend renders report UI

Option B – Heavy Report (Better Architecture)

1. Parent clicks Generate Report
2. Frontend → Backend
3. Backend validates request
4. Backend pushes report job to queue
5. Backend returns:
   "Report is being generated"

6. Worker service:
   - Fetches data from DB
   - Runs AI analysis
   - Generates PDF / structured report
   - Stores report in storage
   - Updates DB status

7. Notification sent to parent
8. Parent downloads finished report
